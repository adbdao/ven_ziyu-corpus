【關鍵字詞解釋】

    結果檔、結果列表、搜尋結果視窗 : 為一您所找到的資料列表視窗.
    原始檔案 : 即您找到的資料的原始檔案.

【程式說明】

    本程式可在文字檔中搜尋所指定之字串.
    但僅能尋找完全相同之字串, 無法使用萬用字元或特殊功能符號.
    如 [ ] * ? ....    

【主要功能】

    1.能正確判斷中文字. (V0.1)
    2.能包含子目錄搜尋. (V0.1)
    3.可跨行搜尋, 並指定忽略行首固定之位元數及行首全半型的空白. (V0.2)
    4.能直接開啟原檔觀看, 並有彩色標示. (V0.1)
    5.配合不同檔案之大小及搜尋結果, 可由使用者決定 [開啟原檔] 及 [顏色處理] 之方式. (V0.2)
    6.在原始檔按下 Esc 鍵或倒退鍵 (BacSpace) , 可以回到上一層之搜尋結果視窗 (V0.2)

【詳細用法】

    1.搜尋設定 : 在主選單中執行 [搜尋] , 以設定搜尋的各種條件, 各項設定說明如下 :

      搜尋字串 : 即您想要搜尋的字串, 本版 (V0.2) 尚未提供萬用字元功能.
      搜尋檔名 : 可使用萬用字元, 如 *.TXT , 但僅能輸入一組.
      起始目錄 : 由左下角的選單來選擇您要搜尋的起始目錄.

      選項 :

        A.包含子目錄 : 可在子目錄中搜尋.
        B.略去換行符號 : 可讓使用者作跨行的字串搜尋.
        C.忽略行首位元 : 有些文章行首有固定的資料, 可指定忽略,
                         例如 CBETA 的經文行首格式如下, 就可指定忽略 19 個字元.

                         T10n0279_p0001b24║　　　　　　　于闐國三藏實叉難陀奉制譯
                         T10n0279_p0001b25║　　世主妙嚴品第一之一
                         T10n0279_p0001b26║如是我聞。一時佛在摩竭提國。阿蘭若法菩


        D.再略去行首全半型空白 : 若文章的行首有全型及半型空白存在時, 指定此項就
                                 可忽略這些空白, 以順利完成跨行的搜尋.

      工作暫存目錄 : 程式執行時暫存檔的目錄, 若不指定, 則以目前目錄為工作目錄.
      系統初值 (按鍵) : 會自動找到系統設定的暫存目錄.

      搜尋 (按鍵) : 按下此鍵即開始搜尋.
      中止 (按鍵) : 按下此鍵即中止搜尋, 並列出目前找到的結果.

    2.搜尋結果視窗 : 找到資料後, 即顯視搜尋結果視窗, 其底色為黃色,
                     主要功能如下 (見主功能表, 在視窗按滑鼠右鍵, 亦有彈出式選單) :

      另存新檔 : 可將找到的結果儲存起來, 本版 (V0.2) 尚不提供載入結果檔的功能.

      開啟原檔 : 可將找到的檔案開啟, 直接找到該行, 並將該行變色處理.
                 開啟的方法有 :

                 i   用滑鼠左鍵雙擊想檢視的該行. (double click)
                 ii  在該行按下 Enter 鍵.
                 iii 在主功能表或彈出式功能表選擇 [開啟原檔].

      開啟原檔的選項 : 原始檔案視窗的底色為白色.

        A.選擇開啟原檔的方式 :

          i  第一種是每次都開啟一個新的視窗, 這種方式適合較小的檔案.
             優點 : 可直接在多個視窗中進行比較.
             缺點 : 若是檔案較大, 則每次都開啟新視窗, 會花費較多的時間載入檔案.

          ii 第二種是每次都進入同一個原始檔案視窗, 這種方式適合較大的檔案.
             優點 : 只要開啟第一次, 之後就省去載入檔案的時間, 大檔尤其好用.
             缺點 : 第二次進入後, 只有該行會變色, 以避免顏色太亂.

        B.開啟原檔時的顏色處理方式 :

          i  只有指定的該行變色, 這個方法適合較大且較多的搜尋結果的檔案.
             優點 : 速度快.
             缺點 : 不易看到同檔其它同樣符合搜尋結果的資料. (沒變色, 所以不易找)

          ii 搜尋到的資料行都要變色, 主要選的那行會變成 [藍色], 其它找到的行會變成 [粉紅色].
             而搜尋字串則變成 [紅色] . 這種方式適合較小的及較少搜尋結果的檔案.
             優點 : 顏色分別很明顯, 在同檔中很容易找到所尋找的字串.
             缺點 : 速度會較慢.

             在每一行逐一變色時, 若想中止, 可使用主功能表上 [停止變色] 的功能.

        C.關閉搜尋結果列表時 (黃色的視窗) , 所有開啟的原始檔一併關閉.
          這個可以省去一一關閉原始檔的麻煩... 尤其您不需要它們的時候.

    3.原始檔案視窗 : 底色為白色的視窗, 即在搜尋結果列表中, 選擇開啟原檔所開啟的視窗.
                     主要功能如下 (見主功能表, 在視窗按滑鼠右鍵, 亦有彈出式選單) :

      跳至某行 : 可直接跳至指定的行數.
      回上層 : 可回到開啟本視窗的結果視窗, 可使用以下方法來達到此目的 :
               Esc 鍵, 後退鍵 (BackSpace) , 或功能表的選項.


【程式缺點】

    1.在搜尋結果放至最大的情況, 若開啟原檔, 則右上角的功能鍵會消失, 原因不明...
      此時要執行主功能表的 [視窗] , 改變之後再放大, 就恢復正常了.
    2.沒有數目統計. (不知有沒有用, 故懶得寫)
    3.尚沒有跨越標記 (如 < > )之搜尋功能. (有空再來想了)
    4.找到的字若有折行則無法順利顯現變色. (太麻煩了, 又怕程式變慢)
    5.....族繁..... (若有重要的事, 請告訴我)

        1. 加快處理速度.
        2. 儲存部份歷史記錄.

           c. 在原始檔按下 ESC 或倒退鍵, 可以回到上一層之搜尋結果視窗.
